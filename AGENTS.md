# AspenBurner 仓库执行约束

版本: v0.5.0

## 当前目标

将本项目从 PowerShell 脚本工具产品化为可交付的专用 Windows 软件，并在本次工作中完成以下闭环：

1. 先评估工程问题、稳定性问题、用户体验和交互逻辑。
2. 先从网上学习并引入合适的 skills，优先覆盖设计、体验、交互、计划与审议能力。
3. 使用 `superpowers` 方法产出设计方案。
4. 使用多模型审议方案，通过后再开始实现。
5. 实现完成后，下载并使用 `opencli` 模拟用户操作进行自审。
6. 通过验证后再视为完成。

## 交付纪律

- 本轮工作默认不与用户进行中途交互，除非出现无法自行发现且高风险的阻塞。
- 所有新增文档统一落到仓库内，优先使用 `docs/` 与 `memory-bank/`。
- 保持向后兼容：现有脚本入口、配置文件和已存在功能不能被无提示破坏。
- 优先做真正影响稳定性、性能和体验的内核产品化，不先做空壳包装。
- 任何完成声明之前必须有新鲜验证证据。

## 实施原则

- 先设计、再计划、后实现。
- 实现阶段坚持 TDD，先写失败测试，再写最小实现。
- 关键设计和实现节点都要有可追溯文档。
- 每完成一个明确工作项，更新 `memory-bank/activeContext.md` 与 `memory-bank/progress.md`。
